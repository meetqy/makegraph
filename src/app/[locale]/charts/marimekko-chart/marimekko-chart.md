## 一句话定义

Marimekko Chart 是一种使用可变宽度堆叠列来展示分类数据的图表。列宽表示类别在整体中的份额，列内各分段高度表示该类别内部子类别的占比，因此它可以同时呈现“类别大小”和“内部构成”。

## 什么是 Marimekko Chart

Marimekko Chart，也常被称为 Mekko Chart，在形式上可以理解为“可变宽度的堆叠柱状图”。普通堆叠柱状图的每个柱子宽度相同，而 Marimekko Chart 的每个柱子宽度会根据该类别的总量或总份额发生变化。

它的两个核心编码方式分别是：

- 列宽：表示该类别在整体中的份额
- 列高内部的各个分段：表示该类别内部子类别的百分比分布

因此，Marimekko Chart 能在一个视图中同时回答两个问题：

- 这个类别有多大？
- 这个类别内部是如何构成的？

## Marimekko Chart 有什么用

Marimekko Chart 最适合用于同时比较“类别规模”和“内部结构”的场景，例如：

- 各地区市场份额，以及每个地区内部的产品结构
- 各品牌销售份额，以及各品牌内部不同渠道的收入占比
- 各部门预算份额，以及每个部门内部不同支出类型的分布
- 各人群样本量，以及每个人群内部答案选择比例
- 各业务单元营收贡献，以及各业务单元内部收入来源拆分

如果你的表达重点不是单纯比较数值大小，而是希望在类别之间建立“规模 + 构成”的双重对比，Marimekko Chart 通常比普通堆叠柱状图更有信息密度。

## 什么时候适合使用 Marimekko Chart

当你的数据满足下面这些条件时，Marimekko Chart 通常非常合适：

- 每个大类都有可比较的总量或总份额
- 每个大类内部还能继续拆分成若干子类别
- 你既关心大类之间的份额差异，也关心组内结构差异
- 你希望把两层信息放在同一张图中展示
- 类别数量适中，通常 3-8 个会比较合适

## 什么时候不适合使用 Marimekko Chart

下面这些场景通常应该优先考虑其他图表：

- 你只关心类别总量大小，用普通柱状图更直接
- 你只关心组内构成，用 100% 堆叠柱状图更清晰
- 类别太多导致很多列过窄，标签和分段都难以识别
- 你需要精确比较很小的差异，数据表、热力图或条形图更适合
- 你要表达连续时间趋势，折线图或堆叠面积图更好

## 与其他图表的区别

| 图表类型 | 主要强调 | 宽度含义 | 高度含义 |
| --- | --- | --- | --- |
| 普通柱状图 | 类别值大小 | 固定 | 数值大小 |
| 堆叠柱状图 | 总量 + 构成 | 固定 | 总量与子项数值 |
| 100% 堆叠柱状图 | 构成比例 | 固定 | 百分比构成 |
| Marimekko Chart | 份额 + 构成 | 类别份额 | 组内百分比 |
| Tree Map | 面积份额 | 无独立轴宽度 | 通过面积表达 |

Marimekko Chart 和 Tree Map 都能表达 part-to-whole，但 Marimekko Chart 更适合有明确分类轴、并且希望保持“从左到右比较类别”的场景。

## 常见问题与风险

### 1. 可读性挑战

- 如果列太多，窄列会让标签难以放置
- 如果子分段太多，内部结构会显得碎
- 如果颜色太接近，图例对应关系会变差

### 2. 精确比较困难

- 非常细小的分段不适合做精确比较
- 宽度与高度同时编码，会提高阅读门槛
- 对不熟悉该图表的读者，需要额外说明图意

### 3. 使用误区

- 把没有总量关系的数据硬塞进 Marimekko Chart
- 列宽使用的总量和列内分段使用的口径不一致
- 在类别过多时仍坚持使用，导致图表过度拥挤

## 适合的典型场景

### 场景 1：市场份额 + 渠道结构

- 不同品牌在整体市场中的份额不同
- 每个品牌内部又有线上、线下、经销商等渠道拆分

### 场景 2：地区份额 + 客户结构

- 各地区对总收入的贡献不同
- 每个地区内部又可拆为企业客户、中小企业、个人客户

### 场景 3：预算份额 + 支出类型

- 各部门占公司总预算的比例不同
- 每个部门内部又可拆为人力、研发、营销、运营等费用

### 场景 4：问卷分组 + 回答分布

- 不同人群组的样本量不同
- 每个组内部又有不同答案占比

### 场景 5：产品组合 + 利润结构

- 各产品线在总销售中的权重不同
- 每条产品线内部又有不同利润来源或成本构成

## 实用建议总结

### 使用 Marimekko Chart 的判断标准

1. 大类总量和组内占比都很重要
2. 类别数不要太多，避免窄列难读
3. 子类别数保持精简，通常 2-5 个最佳
4. 必须保证总量和分段数据口径一致

### 设计建议

- 图例放在图表下方，减少视线跳跃
- 类别标签使用倾斜布局，避免窄列下文字重叠
- 颜色保持区分度，避免太多接近色
- 留出足够底部空间，避免旋转标签被截断

## FAQ 素材

### 1. Marimekko Chart 是什么？

Marimekko Chart 是一种可变宽度堆叠列图。它用列宽表示类别在整体中的份额，用列内分段高度表示该类别内部的百分比构成，因此能同时表达类别大小和内部结构。

### 2. Marimekko Chart 和堆叠柱状图一样吗？

不一样。堆叠柱状图通常所有列等宽，而 Marimekko Chart 会根据类别总量改变列宽，因此额外编码了类别份额信息。

### 3. Marimekko Chart 适合哪些数据？

适合有明确大类总量、且每个大类又能拆成若干子类别的数据，例如市场份额、预算结构、地区收入构成、问卷人群分布等。

### 4. Marimekko Chart 和 mosaic plot 是同一个概念吗？

很多语境下它们被混用，核心都是通过可变宽度矩形来表达分类变量之间的比例关系。具体叫法会因软件和分析背景略有差异。

### 5. 什么时候不该使用 Marimekko Chart？

当类别太多、分段太碎、受众需要精确数值，或者你只关心单一维度比较时，Marimekko Chart 通常不是最佳选择。
