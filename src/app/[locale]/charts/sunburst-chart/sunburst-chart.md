# Sunburst Chart 是什么？

`Sunburst Chart`（旭日图）是一种同时展示**层级关系**与**整体构成**的图表。

它的视觉结构通常是：

- 圆心表示最上层的起点
- 每往外一圈，代表更下一层的分类
- 每个扇区都属于它的父级扇区
- 扇区面积通常对应当前节点的数值大小

如果你的数据天然带有“父级 -> 子级 -> 更细分子级”的关系，旭日图会比普通饼图更容易把结构讲清楚。

---

# 旭日图最适合解决什么问题？

旭日图特别适合这类问题：

- 某个总量由哪些大类组成？
- 每个大类下面又是如何继续拆分的？
- 哪条层级路径贡献最大？
- 哪些子分类集中在同一个父级下面？

相比只看单层占比的图表，旭日图更像是在一个圆形视图里，把“整体 -> 局部 -> 更细局部”的关系一次展示出来。

---

# 常见使用场景

以下场景很适合用 `Sunburst Chart`：

- 销售拆解：`区域 -> 渠道 -> 产品组`
- 流量分析：`来源 -> 活动 -> 落地页`
- 预算结构：`部门 -> 团队 -> 项目`
- 商品目录：`一级类目 -> 二级类目 -> SKU 家族`
- 组织架构：`事业部 -> 团队 -> 成员组`
- 资产构成：`资产类别 -> 子类别 -> 持仓组`

这类数据的共同点是：不仅要看占比，还要看路径和层级结构。

---

# 什么时候优先使用旭日图？

适合优先选择 `Sunburst Chart` 的情况：

- 数据天然有清晰的父子层级
- 层级通常在 `2-4 层` 之间
- 你既想看整体占比，也想看拆分路径
- 你希望读者从中心向外逐层理解数据结构
- 你想在一张图里同时呈现“谁大”和“怎么拆”

如果你的目标不只是回答“谁最多”，而是还要回答“它下面是怎么构成的”，旭日图通常很合适。

---

# 哪些情况不太适合？

以下情况通常不建议优先使用 `Sunburst Chart`：

- 只有单层分类，没有层级关系
- 分类太多、层级太深
- 标签很长，外圈空间不够
- 需要精确比较非常接近的数值
- 用户更关心排序或排名，而不是结构路径

这时通常可以优先考虑：

- `Pie Chart`
- `Doughnut Chart`
- `Bar Chart`
- `Tree Map Chart`
- `Table`

---

# Sunburst Chart 和 Tree Map Chart 的区别

## 共同点

这两类图都适合展示：

- 层级结构
- 整体与部分关系
- 多级分类数据

## 核心区别

- `Sunburst Chart`：用同心圆扇区表达层级，强调**路径关系**
- `Tree Map Chart`：用矩形嵌套表达层级，强调**面积比较**和**空间利用率**

## 什么时候选旭日图？

更适合选 `Sunburst Chart` 的情况：

- 你想强调父子层级是如何一层层展开的
- 你希望读者顺着路径理解结构
- 你想让“从整体到局部”的逻辑更直观

## 什么时候选 Tree Map？

更适合选 `Tree Map Chart` 的情况：

- 你更关心谁大谁小
- 你需要更高的空间利用率
- 分类很多，希望尽量压缩在一个紧凑版面里

---

# 旭日图和饼图 / 环形图有什么区别？

## 饼图或环形图

更适合：

- 展示单层占比
- 分类数量不多
- 只需要回答“总量由哪些部分组成”

## 旭日图

更适合：

- 展示多层级占比
- 同时表达父子关系
- 解释从大类到子类的拆分路径

## 一个简单判断方法

- 只有一层分类：优先考虑 `Pie Chart` 或 `Doughnut Chart`
- 有父子关系、存在多层拆分：优先考虑 `Sunburst Chart`

---

# 一个容易理解的例子

假设你有一组营收数据：

- 第一层：`Total Revenue`
- 第二层：`North America / Europe / Asia Pacific`
- 第三层：`Retail / Online / Partners`
- 第四层：`Electronics / Home / Subscription`

放进 `Sunburst Chart` 之后，你可以很快看出：

- 哪个区域贡献最大
- 哪个区域更依赖某一种渠道
- 某个渠道下的具体业务构成是什么

这也是旭日图最有价值的地方：它不是只告诉你“占比”，而是告诉你“占比来自哪条路径”。

---

# 一句话理解

- `Pie Chart`：看单层占比
- `Tree Map Chart`：看层级面积对比
- `Sunburst Chart`：看层级路径 + 占比结构
